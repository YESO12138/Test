# API 规格文档

### API 接口定义

#### 3.1 用户注册

```
POST /api/v1/auth/register
Content-Type: application/json

Request:
{
    "username": "string (3-20 chars, alphanumeric + underscore)",
    "email": "string (valid email format)",
    "password": "string (8-32 chars, must include: uppercase, lowercase, number)"
}

Response 201:
{
    "code": 201,
    "message": "注册成功，请查收验证邮件",
    "data": {
        "user_id": "bigint",
        "username": "string",
        "email": "string",
        "status": "PENDING"
    }
}

Response 400:
{
    "code": 400,
    "message": "参数验证失败",
    "errors": [
        {"field": "username", "message": "用户名已存在"}
    ]
}
```

#### 3.2 用户登录

```
POST /api/v1/auth/login
Content-Type: application/json

Request:
{
    "login": "string (username or email)",
    "password": "string",
    "remember_me": "boolean (optional, default: false)"
}

Response 200:
{
    "code": 200,
    "message": "登录成功",
    "data": {
        "access_token": "string (JWT, expires in 15min)",
        "refresh_token": "string (JWT, expires in 7d or 30d)",
        "token_type": "Bearer",
        "expires_in": 900,
        "user": {
            "id": "bigint",
            "username": "string",
            "email": "string",
            "status": "ACTIVE"
        }
    }
}

Response 401:
{
    "code": 401,
    "message": "用户名或密码错误",
    "data": {
        "attempts_remaining": 3
    }
}

Response 423:
{
    "code": 423,
    "message": "账户已锁定，请 30 分钟后重试",
    "data": {
        "locked_until": "2024-01-01T12:30:00Z"
    }
}
```


### 错误码规范

| 错误码 | 含义 | 场景 |
|--------|------|------|
| 400 | 参数错误 | 验证失败、格式错误 |
| 401 | 认证失败 | 密码错误、Token 过期 |
| 403 | 权限不足 | 账户禁用、IP 封禁 |
| 409 | 资源冲突 | 用户名/邮箱已存在 |
| 423 | 资源锁定 | 账户锁定 |
| 429 | 请求过多 | 登录频率限制 |
