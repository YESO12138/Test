## 3. CONTRACT (契约锁定)

### API 端点定义

#### 3.1 登录接口

```yaml
POST /api/v1/auth/login
Content-Type: application/json

# Request
{
  "username": "string (required, 3-50 chars)",
  "password": "string (required, 8-100 chars)"
}

# Response 200 - Success
{
  "code": 0,

    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
    "expires_in": 3600,
    "token_type": "Bearer"
  }
}

# Response 401 - Failed
{
  "code": 40101,
  "message": "用户名或密码错误",
  "data": null
}
```

#### 3.2 Token 刷新接口

```yaml
POST /api/v1/auth/refresh
Content-Type: application/json

# Request
{
  "refresh_token": "string (required)"
}

# Response 200
{
  "code": 0,
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "expires_in": 3600,
    "token_type": "Bearer"
  }
}
```

#### 3.3 登出接口

```yaml
POST /api/v1/auth/logout
Authorization: Bearer {access_token}

# Response 200
{
  "code": 0,
  "message": "登出成功"
}
```

### 错误码规范

| 错误码 | HTTP Status | 含义 | 前端处理 |
|--------|-------------|------|----------|
| `40101` | 401 | 用户名或密码错误 | 提示用户重试 |
| `40102` | 401 | Token 已过期 | 尝试刷新 Token |
| `40103` | 401 | Refresh Token 无效 | 强制跳转登录页 |
| `40301` | 403 | 账户已被禁用 | 提示联系管理员 |
| `42901` | 429 | 登录尝试过于频繁 | 显示冷却时间 |

### JWT Payload 规范

```json
{
  "sub": "user-uuid",
  "username": "johndoe",
  "type": "access | refresh",
  "iat": 1699999999,
  "exp": 1700003599
}
```

## 5. 风险评估

### 安全风险

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| **密码泄露** | 🔴 高 | bcrypt 哈希 + 传输层 HTTPS |
| **XSS 窃取 Token** | 🔴 高 | Token 存储于 `HttpOnly Cookie` 而非 localStorage |
| **暴力破解** | 🟡 中 | 登录失败 5 次后锁定账户 15 分钟 |
| **JWT 密钥泄露** | 🔴 高 | 密钥从环境变量读取，禁止硬编码 |
| **Token 无法撤销** | 🟡 中 | 实现 Token 黑名单 或 缩短有效期 |

### 技术风险

| 风险 | 等级 | 替代方案 |
|------|------|----------|
| **JWT 无状态导致无法强制登出** | 🟡 中 | 方案A: Redis 黑名单；方案B: 短有效期 + refresh |
| **并发刷新竞态** | 🟢 低 | refresh_token 单次使用，刷新后旧 Token 失效 |

### 性能风险

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| **bcrypt 计算耗时** | 🟡 中 | cost=12 约 250ms，异步处理不阻塞 |
| **登录日志写入延迟** | 🟢 低 | 异步写入，不影响登录响应 |
