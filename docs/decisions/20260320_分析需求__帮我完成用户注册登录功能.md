## 3. CONTRACT (契约锁定)

### API 端点定义

#### 3.1 用户注册
```
POST /api/v1/auth/register

Request:
{
    "username": "string (3-50 chars)",
    "email": "string (valid email)",
    "password": "string (min 8 chars)"
}

Response 201:
{
    "code": 201,
    "data": {
        "user_id": "uuid",
        "username": "string",
        "email": "string"
    }
}

Error 400/409:
{
    "code": 400|409,
    "error": "VALIDATION_ERROR|EMAIL_EXISTS|USERNAME_EXISTS",
    "message": "string"
}
```

#### 3.2 用户登录
```
POST /api/v1/auth/login

Request:
{
    "email": "string",
    "password": "string"
}

Response 200:
{
    "code": 200,
    "data": {
        "access_token": "string",
        "refresh_token": "string",
        "expires_in": 3600,
        "user": {
            "id": "uuid",
            "username": "string",
            "email": "string"
        }
    }
}

Error 401:
{
    "code": 401,
    "error": "INVALID_CREDENTIALS",
    "message": "邮箱或密码错误"
}
```

#### 3.3 Token刷新
```
POST /api/v1/auth/refresh

Request:
{
    "refresh_token": "string"
}

Response 200:
{
    "code": 200,
    "data": {
        "access_token": "string",
        "expires_in": 3600
    }
}
```

#### 3.4 用户登出
```
POST /api/v1/auth/logout

Headers:
    Authorization: Bearer {access_token}

Response 200:
{
    "code": 200,
    "message": "登出成功"
}
```

### 错误码规范

| 错误码 | HTTP状态 | 场景 |
|--------|----------|------|
| VALIDATION_ERROR | 400 | 参数校验失败 |
| EMAIL_EXISTS | 409 | 邮箱已注册 |
| USERNAME_EXISTS | 409 | 用户名已存在 |
| INVALID_CREDENTIALS | 401 | 登录凭证错误 |
| TOKEN_EXPIRED | 401 | Token已过期 |
| TOKEN_INVALID | 401 | Token无效 |

## 5. 风险评估

### 安全风险

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| 密码泄露 | 🔴 高 | bcrypt加密 + 禁止日志记录明文 |
| Token劫持 | 🟡 中 | HTTPS + HttpOnly Cookie + 短有效期 |
| 暴力破解 | 🟡 中 | 登录失败次数限制 + 验证码机制 |
| SQL注入 | 🟢 低 | 参数化查询 + ORM框架 |

### 技术风险

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| Token刷新竞态 | 🟡 中 | 前端请求队列 + 去重机制 |
| 并发注册冲突 | 🟢 低 | 数据库唯一约束 |
| 分布式Session | 🟡 中 | JWT无状态设计 |

### 替代方案

| 方案 | 适用场景 | 权衡 |
|------|----------|------|
| **OAuth2.0** | 需要第三方登录 | 增加复杂度，提升用户体验 |
| **Session+Cookie** | 传统单体应用 | 简单但不适合分布式 |
| **Magic Link** | 无密码场景 | 用户体验好，依赖邮件服务 |
