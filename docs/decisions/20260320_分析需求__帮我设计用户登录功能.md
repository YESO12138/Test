## 2. SCHEMA (数据驱动)

### 核心表结构

```sql
-- 用户表
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    email VARCHAR(100) UNIQUE,
    status ENUM('active', 'locked', 'disabled') DEFAULT 'active',
    failed_login_count INT DEFAULT 0,
    locked_until DATETIME NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_username (username),
    INDEX idx_status (status)
);

-- 登录尝试表（安全审计）
CREATE TABLE login_attempts (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    identifier VARCHAR(100) NOT NULL,
    ip_address VARCHAR(45) NOT NULL,
    user_agent TEXT,
    success BOOLEAN DEFAULT FALSE,
    failure_reason VARCHAR(100),
    attempted_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_identifier_time (identifier, attempted_at),
    INDEX idx_ip_time (ip_address, attempted_at)
);

-- 用户会话表（多端登录）
CREATE TABLE user_sessions (
    id VARCHAR(64) PRIMARY KEY,
    user_id BIGINT NOT NULL,
    device_name VARCHAR(100),
    device_type ENUM('web', 'mobile', 'desktop') DEFAULT 'web',
    ip_address VARCHAR(45),
    remember_me BOOLEAN DEFAULT FALSE,
    last_active_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    expires_at DATETIME NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_user_expires (user_id, expires_at)
);

-- 刷新令牌表
CREATE TABLE refresh_tokens (
    id VARCHAR(64) PRIMARY KEY,
    user_id BIGINT NOT NULL,
    session_id VARCHAR(64),
    token_hash VARCHAR(255) NOT NULL,
    expires_at DATETIME NOT NULL,
    revoked BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_user_token (user_id, token_hash)
);
```


| AUTH_003 | 429 | 请求过于频繁 |

## 5. 风险评估

### 🔴 高风险

| 风险 | 影响 | 缓解策略 |
|------|------|----------|
| **暴力破解攻击** | 账户安全 | IP + 用户双重限制 + CAPTCHA |
| **会话劫持** | Token 泄露 | HttpOnly Cookie + 设备指纹绑定 |
| **重放攻击** | 请求伪造 | nonce + 时间戳验证 |

### 🟡 中风险

| 风险 | 影响 | 缓解策略 |
|------|------|----------|
| **Token 过期中断** | 用户体验 | 无感刷新 + 静默续期 |
| **多端状态不同步** | 数据一致性 | WebSocket 推送 / 短轮询 |
| **数据库压力** | 性能瓶颈 | Redis 缓存热点数据 |

### 替代方案

1. **OAuth 2.0 第三方登录** → 二期扩展
2. **无密码登录（邮箱/短信验证码）** → 提升体验，增加成本
3. **TOTP 双因素认证** → 高安全场景
