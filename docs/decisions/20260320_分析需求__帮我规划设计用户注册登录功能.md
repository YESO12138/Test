## 2. SCHEMA (数据驱动)

### 用户表

```sql
CREATE TABLE users (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username      VARCHAR(50) NOT NULL,
    email         VARCHAR(255) NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    phone         VARCHAR(20),
    status        VARCHAR(20) NOT NULL DEFAULT 'PENDING',
    email_verified BOOLEAN DEFAULT FALSE,
    created_at    TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at    TIMESTAMP NOT NULL DEFAULT NOW(),
    last_login_at TIMESTAMP,
    
    CONSTRAINT uk_users_username UNIQUE (username),
    CONSTRAINT uk_users_email UNIQUE (email),
    CONSTRAINT chk_users_status CHECK (status IN ('PENDING', 'ACTIVE', 'SUSPENDED', 'DELETED'))
);

CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_status ON users(status);
```

### 会话表

```sql
CREATE TABLE sessions (
    id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id       UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    refresh_token VARCHAR(500) NOT NULL,
    device_info   VARCHAR(255),
    ip_address    VARCHAR(45),
    expires_at    TIMESTAMP NOT NULL,
    created_at    TIMESTAMP NOT NULL DEFAULT NOW(),
    
    CONSTRAINT uk_sessions_refresh_token UNIQUE (refresh_token)
);

CREATE INDEX idx_sessions_user_id ON sessions(user_id);
CREATE INDEX idx_sessions_expires_at ON sessions(expires_at);
```

## 5. 风险评估

### 🔴 高风险

| 风险 | 影响 | 缓解方案 |
|------|------|----------|
| **密码存储安全** | 数据泄露导致用户凭证暴露 | 使用 bcrypt (cost≥12)，禁止明文存储 |
| **暴力破解** | 账户被盗用 | 实现登录频率限制 (5次/分钟/IP)，账户锁定机制 |
| **Token 劫持** | 会话被窃取 | HttpOnly Cookie + SameSite，短期 access_token (1h) |

### 🟡 中风险

| 风险 | 影响 | 缓解方案 |
|------|------|----------|
| **并发注册** | 同一邮箱被多次注册 | 数据库唯一约束 + 乐观锁 |
| **Token 过期处理** | 用户体验下降 | 静默刷新机制，过期前 5 分钟自动刷新 |
| **跨域认证** | Cookie 无法携带 | 配置 CORS + Credentials，或使用 Authorization Header |

### 🟢 替代方案

| 场景 | 主方案 | 替代方案 |
|------|--------|----------|
| Token 存储 | localStorage | HttpOnly Cookie (更安全，防 XSS) |
| 认证方式 | JWT | Session + Redis (服务端可控，但需存储) |
| 密码加密 | bcrypt | argon2 (更安全，但需额外依赖) |
