## 2. SCHEMA (数据驱动)

### 用户表

```sql
CREATE TABLE users (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username        VARCHAR(50) NOT NULL UNIQUE,
    email           VARCHAR(255) NOT NULL UNIQUE,
    password_hash   VARCHAR(255) NOT NULL,
    status          VARCHAR(20) NOT NULL DEFAULT 'ACTIVE' 
                    CHECK (status IN ('ACTIVE', 'INACTIVE', 'LOCKED', 'PENDING')),
    login_attempts  INTEGER NOT NULL DEFAULT 0,
    locked_until    TIMESTAMP WITH TIME ZONE,
    last_login_at   TIMESTAMP WITH TIME ZONE,
    last_login_ip   VARCHAR(45),
    created_at      TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW()
);

-- 索引策略
CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_status ON users(status);
```

### 会话表

```sql
CREATE TABLE sessions (
    id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id        UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    token_hash     VARCHAR(255) NOT NULL UNIQUE,
    refresh_token  VARCHAR(255) UNIQUE,
    ip_address     VARCHAR(45),
    user_agent     TEXT,
    expires_at     TIMESTAMP WITH TIME ZONE NOT NULL,
    created_at     TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_sessions_user_id ON sessions(user_id);
CREATE INDEX idx_sessions_expires_at ON sessions(expires_at);
```

### 登录日志表

```sql
CREATE TABLE login_logs (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id     UUID REFERENCES users(id) ON DELETE SET NULL,
    username    VARCHAR(50) NOT NULL,
    ip_address  VARCHAR(45) NOT NULL,
    user_agent  TEXT,
    status      VARCHAR(20) NOT NULL CHECK (status IN ('SUCCESS', 'FAILED', 'LOCKED')),
    fail_reason VARCHAR(100),
    created_at  TIMESTAMP WITH TIME ZONE NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_login_logs_user_id ON login_logs(user_id);
CREATE INDEX idx_login_logs_created_at ON login_logs(created_at);
```

## 3. CONTRACT (契约锁定)

### API 规格定义

#### POST /api/v1/auth/login

**请求:**
```json
{
  "username": "string (required, 3-50字符)",
  "password": "string (required, 8-100字符)",
  "remember_me": "boolean (optional, 默认false)"
}
```

**响应 (200 OK):**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "Bearer",
    "expires_in": 3600,
    "user": {
      "id": "uuid",
      "username": "string",
      "email": "string",
      "status": "ACTIVE"
    }
  }
}
```

**错误响应:**
```json
{
  "code": 40101,
  "message": "用户名或密码错误",
  "data": {
    "remaining_attempts": 3
  }
}
```

#### POST /api/v1/auth/refresh

**请求:**
```json
{
  "refresh_token": "string (required)"
}
```

**响应 (200 OK):**
```json
{
  "code": 0,
  "data": {
    "access_token": "new_token",
    "expires_in": 3600
  }
}
```

#### POST /api/v1/auth/logout

**请求头:** `Authorization: Bearer {token}`

**响应 (200 OK):**
```json
{
  "code": 0,
  "message": "登出成功"
}
```

### 错误码规范

| 错误码 | HTTP状态 | 含义 | 前端处理 |
|--------|----------|------|----------|
| 40101 | 401 | 用户名或密码错误 | 显示错误，剩余次数 |
| 40102 | 401 | 账户已锁定 | 显示解锁时间 |
| 40103 | 401 | Token已过期 | 尝试刷新Token |
| 40104 | 401 | Token无效 | 跳转登录页 |
| 42901 | 429 | 请求过于频繁 | 显示冷却时间 |

## 5. 风险评估

### 安全风险

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| 密码明文传输 | 高 | 强制HTTPS，前端不加密(避免假安全感) |
| 暴力破解 | 高 | 登录失败锁定 + IP限流 + 验证码(可选) |
| Token泄露 | 中 | HttpOnly Cookie + 短有效期 + Refresh Token轮换 |
| Session固定攻击 | 中 | 登录成功后重新生成Session ID |

### 技术风险

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| JWT无法主动失效 | 中 | 使用短有效期 + 黑名单机制(可选) |
| 并发刷新Token | 低 | Refresh Token单次使用 + 轮换机制 |

### 替代方案

| 场景 | 主方案 | 替代方案 | 选择理由 |
|------|--------|----------|----------|
| Token存储 | HttpOnly Cookie | localStorage | 防XSS攻击 |
| 会话管理 | JWT无状态 | Session+Redis | 支持水平扩展 |
| 密码哈希 | bcrypt | Argon2 | 成熟稳定，足够安全 |
