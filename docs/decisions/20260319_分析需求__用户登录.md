## 2. SCHEMA (数据驱动)

### 用户表设计

```sql
-- 用户主表
CREATE TABLE users (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email           VARCHAR(255) UNIQUE NOT NULL,
    username        VARCHAR(50) UNIQUE,
    password_hash   VARCHAR(255) NOT NULL,
    status          VARCHAR(20) DEFAULT 'active' CHECK (status IN ('active', 'inactive', 'locked')),
    login_attempts  INTEGER DEFAULT 0,
    created_at      TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    last_login_at   TIMESTAMP WITH TIME ZONE
);

-- 索引策略
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_status ON users(status);

-- 登录日志表（审计需求）
CREATE TABLE login_logs (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id     UUID REFERENCES users(id),
    ip_address  VARCHAR(45),
    user_agent  TEXT,
    status      VARCHAR(20) CHECK (status IN ('success', 'failed')),
    created_at  TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_login_logs_user_id ON login_logs(user_id);
CREATE INDEX idx_login_logs_created_at ON login_logs(created_at);
```

## 3. CONTRACT (契约锁定)

### API 规格书

#### 3.1 登录接口

```yaml
POST /api/v1/auth/login
Content-Type: application/json

# 请求体
{
  "email": "user@example.com",    # 必填，邮箱格式
  "password": "string"            # 必填，6-32字符
}

# 成功响应 (200)
{
  "code": 0,
  "data": {
    "user": {
      "id": "uuid",
      "email": "user@example.com",
      "username": "string"
    },
    "tokens": {
      "access_token": "jwt_string",
      "refresh_token": "jwt_string",
      "expires_in": 3600
    }
  }
}

# 错误响应
# 400: 参数校验失败
# 401: 凭证无效
# 423: 账户锁定
{
  "code": 40101,
  "message": "邮箱或密码错误",
  "data": null
}
```

#### 3.2 Token 刷新接口

```yaml
POST /api/v1/auth/refresh
Content-Type: application/json
Authorization: Bearer {refresh_token}

# 成功响应 (200)
{
  "code": 0,
  "data": {
    "access_token": "jwt_string",
    "expires_in": 3600
  }
}

# 错误响应
# 401: Token 无效或过期
```

#### 3.3 登出接口

```yaml
POST /api/v1/auth/logout
Authorization: Bearer {access_token}

# 成功响应 (200)
{
  "code": 0,
  "message": "登出成功"
}
```

### 错误码规范

| 错误码 | HTTP 状态码 | 说明 |
|--------|-------------|------|
| 40101 | 401 | 邮箱或密码错误 |
| 40102 | 401 | Token 无效 |
| 40103 | 401 | Token 已过期 |
| 42301 | 423 | 账户已锁定 |
| 40001 | 400 | 参数校验失败 |

## 5. 风险评估

### 安全风险

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| 密码泄露 | 🔴 高 | 使用 bcrypt/scrypt 加密，禁止明文存储 |
| 暴力破解 | 🔴 高 | 登录失败次数限制 + IP 限流 + CAPTCHA |
| Token 劫持 | 🟡 中 | HttpOnly Cookie + Secure Flag + SameSite |
| 重放攻击 | 🟡 中 | Token 中加入 jti (JWT ID) + 黑名单机制 |

### 技术风险

| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| Token 过期处理不当 | 🟡 中 | 前端实现自动刷新 + 401 拦截器 |
| 并发登录问题 | 🟢 低 | 明确业务规则：允许多设备登录或单设备登录 |
| 密码重置流程缺失 | 🟡 中 | 作为独立需求后续迭代 |

### 替代方案

| 场景 | 主方案 | 替代方案 |
|------|--------|----------|
| 认证方式 | JWT | Session + Cookie (更简单，但需要服务端存储) |
| 密码加密 | bcrypt | Argon2 (更安全，但需要额外依赖) |
| 前端状态管理 | Pinia/Vuex | React Context / Zustand |
